Volumetric microscopy datasets routinely exceed the memory of a single GPU and the patience of a single user, making neural-network inference a workflow problem as much as a modelling problem. We present \cellmapflow{}, an open-source framework that exposes deep-learning predictions over multi-terabyte volumes as virtual Zarr arrays, computed on demand and streamed directly into Neuroglancer for interactive exploration. The same execution engine drives whole-volume batch inference on a cluster, using a Daisy-based scheduler to distribute blocks across LSF-managed GPU workers. \cellmapflow{} accepts models in PyTorch TorchScript, DaCapo, BioImage.io, HuggingFace Hub, and lab-internal formats, reads Zarr/N5/OME-Zarr inputs from local disk, S3, or precomputed cloud buckets via TensorStore, and composes pluggable input normalizers and output post-processors (thresholding, mutex watershed, label cleanup) into YAML-defined pipelines. Multiple models can be fused at inference time using AND/OR/consensus merge modes, supporting model comparison and ensembling without offline export. We describe the system architecture, demonstrate use cases drawn from the CellMap segmentation pipeline, and report benchmarks of interactive latency, blockwise throughput, and cluster scaling. \cellmapflow{} is released under a BSD-3 license at \url{https://github.com/janelia-cellmap/cellmap-flow}.
